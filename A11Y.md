# Accessibility Features - Barakah Lanterns

## Color Contrast
- Background colors: Deep navy (#0a1428) with WCAG AA compliant contrast ratios
- Text colors: High contrast light text on dark backgrounds (minimum 7:1 ratio)
- Interactive elements: Gold (#f59e0b) and emerald (#10b981) with sufficient contrast

## Keyboard Navigation
- All interactive elements are keyboard accessible
- Navigation uses semantic HTML with proper link structure
- Modal dialogs are keyboard-navigable with Escape to close
- Tab order is logical and follows visual flow

## Screen Reader Support
- Semantic HTML elements: `<main>`, `<header>`, `<nav>`
- ARIA labels on interactive components
- Alt text descriptions for icons and images
- Form labels properly associated with inputs
- Status updates announced to screen readers via ARIA live regions

## Motion & Animations
- Framer Motion animations respect `prefers-reduced-motion` settings
- All animated interactions have non-animated alternatives
- Loading states are clearly indicated with text and icons
- Transitions use appropriate timing and easing

## Input & Forms
- Form fields have clear, associated labels
- Required fields are marked
- Error messages are descriptive and linked to inputs
- Character counts and feedback provided in real-time
- Touch targets are minimum 44x44px for mobile accessibility

## Navigation
- Bottom navigation is persistent and always accessible
- Current page is indicated in navigation
- Navigation uses clear, descriptive labels with icons
- Mobile-first responsive design ensures usability on all devices

## Content Structure
- Heading hierarchy is logical and proper (no skipped levels)
- Lists use semantic `<ul>` and `<ol>` elements
- Important information is not conveyed by color alone
- Content is readable in high contrast mode

## Focus Indicators
- Focus visible indicators on all interactive elements
- Focus ring uses contrasting colors (gold accent)
- Focus management in modals and dialogs
- Skip links available for keyboard users

## Testing Recommendations
1. Test with keyboard navigation only
2. Use screen readers (NVDA, JAWS, VoiceOver)
3. Check with browser zoom at 200%
4. Test with Windows High Contrast mode
5. Validate with WCAG 2.1 Level AA tools
6. Test with color blindness simulators
7. Verify touch targets on mobile devices
8. Check motion settings on different devices

## Continuous Improvement
- Regular accessibility audits recommended
- User feedback from accessibility community
- Updates to follow WCAG 2.2 guidelines as needed
- Annual review of accessibility features
